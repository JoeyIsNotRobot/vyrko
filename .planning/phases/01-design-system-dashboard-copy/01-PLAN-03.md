    - Div raiz: x-data="loadingModal()" x-show="show" x-transition:enter e x-transition:leave básicos

      - Título: <p class="text-white font-bold text-lg text-center">Processando...</p>

        Progress fill: <div class="h-2 rounded-full bg-gradient-to-r from-blue-600 to-emerald-500 transition-all duration-500" :style="'width: ' + progress + '%'"></div>
        </div>

      - Step text: <p class="text-sm text-slate-400 mt-3 text-center" x-text="currentStep"></p>

      - Error block (x-show="error"):
        <div x-show="error" class="bg-red-900/40 border border-red-500/50 rounded-lg p-4 mt-4 text-center">
            <p class="text-red-300 text-sm" x-text="error"></p>
            <button type="button" class="btn danger mt-3" @click="error = null; progress = 0; show = false">
                Tentar novamente
            </button>
        </div>

    NOTAS TÉCNICAS:
    - NÃO usar x-cloak aqui (evita flash se Alpine não carregou a tempo — mas para um modal oculto por padrão, x-show="show" onde show=false é suficiente)
    - x-transition simples: x-transition:enter="transition ease-out duration-200" x-transition:enter-start="opacity-0" x-transition:enter-end="opacity-100" x-transition:leave="transition ease-in duration-150" x-transition:leave-start="opacity-100" x-transition:leave-end="opacity-0"
    - O Blade component é stateless (sem props) — todo estado é gerenciado pelo Alpine data object
    - Para chamar o modal de JS externo: acessar via window.Alpine.$data(document.querySelector('[x-data="loadingModal()"]')) ou expor via window.loadingModal em app.js

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Alpine modal ↔ chamadores JS | O modal expõe API via Alpine store ou window — qualquer JS da página pode abrir/fechar |
| datalist ↔ input value | Valor final do input é texto livre — server-side validation deve aceitar qualquer string |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | loading-modal error text (x-text) | mitigate | x-text escapa HTML automaticamente no Alpine — mensagens de erro não podem injetar HTML |
| T-03-02 | Denial of Service | ESC bloqueado durante modal ativo | accept | ESC é bloqueado apenas quando show=true e error=null; no estado de erro o usuário pode clicar "Tentar novamente" — não há lock infinito |
| T-03-03 | Tampering | datalist options — XSS via input | accept | datalist são opções estáticas hardcoded no Blade — sem input de usuário nas options; valor do input é validado server-side no CandidateProfileController existente |
| T-03-SC | Tampering | npm/pip installs | accept | Plano 03 não instala novos pacotes |
</threat_model>
