    ```
    <div
        class="import-card"
        x-data="{
            file: null,
            dragOver: false,
            fileError: null,
            importSuccess: false,
            setFile(f) {
                const allowed = ['pdf', 'docx', 'txt'];
                const ext = f.name.split('.').pop().toLowerCase();
                if (!allowed.includes(ext)) {
                    this.fileError = 'Formato não suportado. Use um arquivo PDF, DOCX ou TXT.';
                    this.file = null;
                    return;
                }
                this.fileError = null;
                this.file = f;
                const dt = new DataTransfer();
                dt.items.add(f);
                $refs.fileInput.files = dt.files;
            },
            handleDrop(e) {
                e.preventDefault();
                this.dragOver = false;
                const f = e.dataTransfer?.files?.[0];
                if (f) this.setFile(f);
            }
        }"
        @import:success.window="importSuccess = true; setTimeout(() => importSuccess = false, 8000)"
    >
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /career/import | Upload de arquivo não-autenticado bloqueado por middleware auth+verified |
| Drop zone JS → input[name=resume] | Extensão validada no frontend (UX) E no backend (ResumeImportRequest — fonte de verdade) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Drop zone: DataTransfer injeta File em input | accept | Frontend valida extensão para UX; ResumeImportRequest valida mimes:pdf,txt,docx no servidor — dupla camada já existente |
| T-02-02 | Spoofing | Accept attribute ".pdf,.docx,.txt" no input | accept | Atributo é hint de UI; validação real é backend. Bypass é intencional (usuário pode abrir qualquer arquivo via dev tools) |
| T-02-03 | Information Disclosure | Nome do arquivo exibido na drop zone | accept | Nome do arquivo é dado do próprio usuário, sem risco de disclosure |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | Nenhum pacote instalado neste plano |
</threat_model>
